Rev 2.0 introduced USB-C flashing by switching from the ESP32 to the ESP32-S2. This eliminates the need for a separate USB-to-serial adapter.

![Rev 2.0 board](/images/board_rev2.0.jpg)

## Specifications

- **MCU:** ESP32S2-Mini-2
- **Board platform:** esp32-s2-saola-1
- **Power:** DC Input (12V, 5.5x2.1mm)
- **Flashing:** USB-C port
- **Fan outputs:** 4x PWM headers
- **Status LED:** Red (single)
- **Sensor:** HDC1080 Temperature and Humidity
- **Buttons:** Reset, Boot, 3x User
- **Expansion:** I2C port (100mil, SCL, SDA, INT)

<Aside type="caution" title="USB not working?">
The ESP32-S2 has known USB reliability issues. If the flash button doesn't detect your board:
1. Try a different USB cable (data cable, not charge-only)
2. Hold BOOT button while plugging in USB
3. Use [ESPWEBTOOL](https://esptool.spacehuhn.com/) as an alternative web flasher
4. As last resort, use `esptool.py` CLI: `esptool.py --chip esp32s2 write_flash 0x0 firmware.bin`

[Download prebuilt binary](/firmware/fancontroller-r2-0-esp32s2.factory.bin)
</Aside>

<Aside type="caution" title="USB-C is for flashing only">
The USB-C port provides power to the ESP32-S2 microcontroller only. It cannot power the fans. Fans require an external 12V DC power supply connected to the DC input jack (5.5x2.1mm).
</Aside>

## I2C Extension Port

The board offers an I2C extension port above Fan 4.

| Nr. | Pin | Description |
|-----|-----|-------------|
| 1 | GND | |
| 2 | INT | GPIO 35 |
| 3 | SCL | GPIO 34, 4.7k Pull-Up |
| 4 | SDA | GPIO 33, 4.7k Pull-Up |
| 5 | +3V3 | |
