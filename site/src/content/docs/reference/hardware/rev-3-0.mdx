Rev 3.0 added Qwiic I2C connectivity, a NeoPixel expansion port, PWM signal buffering for improved fan signal integrity, and a bottom SMD expansion header. Unlike Rev 3.1+, it does not have RGB status LEDs.

## Specifications

- **MCU:** ESP32S2-Mini-2
- **Board platform:** esp32-s2-saola-1
- **Power:** DC Input (12V, 5.5x2.1mm)
- **Flashing:** USB-C port
- **Fan outputs:** 4x PWM headers with signal buffering
- **Status LED:** Red (single)
- **Sensor:** HDC1080 Temperature and Humidity
- **Buttons:** Reset, Boot, 3x User
- **Expansion:** QWIIC I2C, I2C port (100mil), NeoPixel port (5V, max 2A), SMD expansion header

<Aside type="note" title="NeoPixel framework requirement">
The NeoPixel port requires the `arduino` framework, since GPIO42 requires use of the `fastled_clockless` platform, which is not supported on `esp-idf`.
</Aside>

## Factory Firmware Installation

<FlashButton manifest="/firmware/fancontroller-r3-0.manifest.json" label="Flash Rev 3.0 Factory Firmware" />

<Aside type="caution" title="USB not working?">
The ESP32-S2 has known USB reliability issues. If the flash button doesn't detect your board:
1. Try a different USB cable (data cable, not charge-only)
2. Hold BOOT button while plugging in USB
3. Use [ESPWEBTOOL](https://esptool.spacehuhn.com/) as an alternative web flasher
4. As last resort, use `esptool.py` CLI: `esptool.py --chip esp32s2 write_flash 0x0 firmware.bin`

[Download prebuilt binary](/firmware/fancontroller-r3-0-esp32s2.factory.bin)
</Aside>

<Aside type="caution" title="USB-C is for flashing only">
The USB-C port provides power to the ESP32-S2 microcontroller only. It cannot power the fans. Fans require an external 12V DC power supply connected to the DC input jack (5.5x2.1mm).
</Aside>
